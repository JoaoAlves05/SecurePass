Deep Dive: The Vault

Transparent, local-first security architecture.

Local-First Data Processing

Unlike traditional password managers that immediately sync your data to the cloud, SecurePass processes everything locally on your device.

  • Encryption

    Data is encrypted using AES-256-GCM before it ever touches storage.

  • Key Derivation

    Your master password is transformed into an encryption key using PBKDF2 with 100,000 iterations.

Deep Dive: The Analyzer

Understanding how we measure password strength.

Entropy vs. Length

Length is important, but randomness (entropy) is key. "Password123" is long but weak. "Tr0ub4dor&3" is shorter but stronger. We use the zxcvbn algorithm to calculate bits of entropy, detecting common patterns, names, and keyboard walks.

k-Anonymity

To check if your password was leaked without revealing it, we hash it with SHA-1. We send the first 5 characters (prefix) to the API. The API returns all hashes starting with that prefix. We then check locally if your full hash is in that list. The server never sees your password.

Crack Time Estimation

We estimate how long it would take a supercomputer to crack your password based on its entropy. This gives you a realistic understanding of your security posture.

Privacy & Anonymity

We protect your identity, even when checking for breaches.

k-Anonymity Model

When checking if a password is compromised, we never send the full password. We hash it (SHA-1) and send only the first 5 characters to the HaveIBeenPwned API.

Zero-Knowledge

We don't know your master password. If you lose it, your data is lost forever. This ensures that no one (including us) can access your vault.

Open Source

Our code is public. Security researchers and the community can audit every line of code to ensure there are no backdoors.

Frequently Asked Questions

Where are my passwords stored?

Your passwords are stored exclusively in your browser's local storage, encrypted with your master password. They are never sent to our servers.

What if I lose my master password?

Because we use a Zero-Knowledge architecture, we cannot recover your master password. If you lose it, your data is lost. This ensures that no one but you can access your vault.

Is it really free?

Yes, SecurePass is 100% free and open source. There are no premium plans or hidden fees.

How does the breach check work?

We use k-Anonymity. We hash your password and send only the first 5 characters of the hash to the Have I Been Pwned API. This returns a list of potential matches, which we check locally. The API never sees your full hash.
